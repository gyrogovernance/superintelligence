# Export 16: `external/llama.cpp/tests/test-quantize-perf.cpp`

## What it tests

**Micro-benchmarks** for quantization-related ops on synthetic buffers: flags include **`op_quantize_row_q`**, **`op_dequantize_row_q`**, **`op_vec_dot_q`**, etc. (`quantize_perf_params` ~31+). Uses **`rdtsc`** / **`rdtscp`** on x86 for cycle timing; **`QK`** **32** aligned sizes (`L1_SIZE`, `L2_SIZE`, ...).

## Q8_0 vec_dot

- When **`op_vec_dot_q`** is enabled, benchmarks **`vec_dot`** paths for included types (CLI filters types via **`include_types`**). Q8_0 appears if listed in the run configuration (same trait pointers as runtime).

## Relation to hooks

- Measures **throughput**, not correctness. A gyro hook that adds overhead will show here vs stock CPU build; use same binary with **`GGML_GYROSCOPIC_MATMUL=0` vs `1`** for A/B.

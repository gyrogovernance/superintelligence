# Export 9: `external/llama.cpp/ggml/src/ggml-gyroscopic/arch/x86/repack.cpp`

## Role relative to export 8

- Large x86 translation unit included when building the CPU backend; provides **AVX / AVX2 / AVX512** implementations for **repack** quantization helpers and **GEMV/GEMM** templates used when **`tensor_traits` + arch** select a path that compiles to these symbols.
- **Q8_0 x Q8_0** `ggml_gemm_q8_0_*` **optimized symbols are not defined here** (grep shows no `void ggml_gemm_q8_0` definitions in this file). For x86, **`ggml_gemm_q8_0_*` resolve to generic implementations in `repack.cpp`** (scalar nested loops over `block_q8_0x4`, ~2281+).

## Q8_0-adjacent hot path (AVX2): Q4 (and similar) x Q8_0

- Under `#if defined(__AVX2__) || defined(__AVX512F__)`, templates such as **`gemv_q4_b32_8x8_q8_0_lut_avx`** (~518+): RHS **interleaved 4-bit blocks** (`block_tx8` e.g. `block_q4_0x8`), LHS **`block_q8_0`** rows.
- Pattern (~544-620): outer over Q8_0 blocks; **load 256-bit** RHS quants, **nibble expand** via LUT/shuffle to signed int8; load **two 128-bit** halves of Q8_0 `qs`, broadcast/permutations; **`mul_sum_i8_pairs_acc_int32x8`** (and related) accumulate int32 lanes; **fp16 scales** (`GGML_CPU_FP16_TO_FP32`) applied; horizontal combine to **float** row accumulator (`acc_row`), then store.
- **32-wide:** inner block still tied to **32** int8 per `QK8_0` block semantics in comments ("Process Q8_0 blocks one by one").
- **8-wide column tiling:** processes **8** interleaved column blocks per inner x step (`nc / 8`).

## Microkernel grammar (vs `arch/x86/quants.c` vec_dot)

- **Same family:** int8 dot via **widening / madd**-style helpers (`mul_sum_i8_pairs_acc_int32x8` etc., not identical to `mul_sum_i8_pairs_float` in quants dot).
- **Different orchestration:** repack **interleaved** block layouts (`block_q4_0x8`, packed `qs` lanes), heavy **shuffle / blend** to align 4-bit decoded bytes with Q8_0 rows; quants `ggml_vec_dot_q8_0_q8_0` uses **pair of plain `block_q8_0`** loads per block index.

## Tile / blocking (from sampled GEMV)

- **nr** rows of Q8_0; **nc/8** column super-blocks; inner **`nb = n/32`** blocks along K.

- **None** in the scanned Q4xQ8_0 AVX2 GEMV region.

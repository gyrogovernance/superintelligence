## Target ops: where implementation lives and path

**`GGML_OP_MUL_MAT` / `GGML_OP_MUL_MAT_ID`**

- **Not** implemented in `ops.cpp`. Implemented in `ggml-cpu.c`: `ggml_compute_forward_mul_mat`, `ggml_compute_forward_mul_mat_id` (+ `*_one_chunk`). `ops.cpp` only **calls** `ggml_compute_forward_mul_mat` from a helper that builds synthetic tensors (around line 6644) for another feature (GEMM wrapper path).

**`GGML_OP_OUT_PROD`**

- `ggml_compute_forward_out_prod` in `ops.cpp` (~4331-4378). Dispatch on `src0->type`:
  - Quantized list (includes `GGML_TYPE_Q8_0`, MXFP4, NVFP4, many K-quants, etc.) -> `ggml_compute_forward_out_prod_q_f32`: dequant row via `ggml_get_type_traits(type)->to_float` into per-thread `wdata` scratch, then `ggml_vec_mad_f32` on float.
  - `GGML_TYPE_F32` -> `ggml_compute_forward_out_prod_f32`: parallel over `nr = ne1*ne2*ne3`, tiles `blck_0 = MAX(GGML_VEC_MAD_UNROLL, 32)`, `blck_1 = 16`, inner loop `ggml_vec_mad_f32_unroll` or `ggml_vec_mad_f32`.
  - `GGML_TYPE_F16` -> `GGML_ABORT` (todo).
- **Exactness:** quantized path: dequant to **float** then MAD -> **float** accumulation. F32 path: **float**.

**`GGML_OP_SOFT_MAX`**

- `ggml_compute_forward_soft_max` (~5348-5363): `src0->type == GGML_TYPE_F32` -> `ggml_compute_forward_soft_max_f32`; else abort.
- **Exactness:** **float** (exp, sum, scale).

**`GGML_OP_RMS_NORM`**

- `ggml_compute_forward_rms_norm` -> `ggml_compute_forward_rms_norm_f32` only in switch (~3768+). **float** path.

**`GGML_OP_ROPE`**

- `ggml_compute_forward_rope` (~5879-5898): `F16` -> `ggml_compute_forward_rope_flt<ggml_fp16_t>`; `F32` -> `ggml_compute_forward_rope_flt<float>`; else abort.

**`GGML_OP_UNARY`**

- `ggml_compute_forward_unary` (~9674+): reads `ggml_get_unary_op(dst)`; `switch (op)` dispatches to `ggml_compute_forward_gelu`, `_silu`, `_relu`, `_tanh`, `_exp`, `_gelu_erf`, `_gelu_quick`, `_hardswish`, `_hardsigmoid`, `_floor`, `_ceil`, etc. (full list in source switch ~9680-9770+).

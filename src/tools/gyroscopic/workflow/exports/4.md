## `ggml_backend_cpu_get_extra_buffer_types`

- Static vector built once: may append **AMX** buft (if `__AMX_INT8__` && `__AVX512VNNI__`), **RISC-V SpaceMIT**, **KleidiAI**, **`GGML_USE_CPU_REPACK`** -> `ggml_backend_cpu_repack_buffer_type()`.
- x86 AVX2-only focus: **repack** is the relevant extra buft when `GGML_USE_CPU_REPACK` is on at build time.

## `ggml_backend_cpu_is_extra_buffer_type` / `supports_buft`

- Host buft or any registered extra buft => supported for allocation.

## `ggml_backend_cpu_device_supports_op` (matmul-relevant)

- Ops `NONE`, `RESHAPE`, `VIEW`, `PERMUTE`, `TRANSPOSE`: true.
- **Extra buffer:** first up to **4** `src[i]` checked; if buffer `buft` is extra, delegates to `buf_extra->supports_op(dev, op)` (**repack/AMX-specific** logic). **Bypass:** can declare support independent of generic CPU switch.

**`GGML_OP_MUL_MAT`**

- `src1->type == GGML_TYPE_F32` **or** `src1->type == ggml_get_type_traits_cpu(src0->type)->vec_dot_type` (e.g. Q8_0 activations for Q8 weight).

**`GGML_OP_OUT_PROD`**

- `src0` is F32 **or** (`ggml_is_quantized(src0->type)` and `src0->ne[2]==src1->ne[2]` and `src0->ne[3]==src1->ne[3]`); `src1` is F32; `op->type == F32`.

**`GGML_OP_MUL_MAT_ID`**

- Falls through `default: return true` (no special case in the shown switch).

Other cases in switch: `CPY` / `SET_ROWS` IQ exclusions, `SOFT_MAX_BACK`, `IM2COL_BACK`, `GET_ROWS_BACK` (not detailed here).

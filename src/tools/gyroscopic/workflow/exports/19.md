# Export 19: `external/llama.cpp/ggml/src/ggml-gyroscopic/traits.cpp`

## What this file is

The **extra-buffer dispatch loop** only. It does **not** register `type_traits_cpu` or `vec_dot` pointers; those live in **`ggml-cpu.c`** (`exports/3.md`).

## `ggml_cpu_extra_compute_forward`

- Iterates `ggml_backend_cpu_get_extra_buffer_types()` (from `ggml-cpu.cpp`: repack when `GGML_USE_CPU_REPACK`, plus AMX / Kleidi / SpaceMIT when enabled).
- For each non-null `extra` with `context`, casts to `ggml::cpu::extra_buffer_type`, calls `buf_extra->get_tensor_traits(op)`.
- If `tensor_traits && tensor_traits->compute_forward(params, op)` returns true, returns **true** (caller **skips** generic `ggml_compute_forward` switch).

## `ggml_cpu_extra_work_size`

- Same iteration pattern; if `tensor_traits->work_size(n_threads, op, *size)` returns true, propagates.

## Wiring diagram (MUL_MAT)

1. Graph node `GGML_OP_MUL_MAT` enters `ggml_compute_forward` (`ggml-cpu.c`).
2. **This file runs first:** if a registered extra buffer’s `get_tensor_traits` returns a traits object whose `compute_forward` handles the op (e.g. repack `tensor_traits` for `MUL_MAT`), **done** -- no `ggml_compute_forward_mul_mat`, no gyro GEMM block, no `vec_dot` from generic path.
3. If no trait handles it, fall through to generic mul_mat (gyro GEMM / vec_dot as in export 3).

## Q8_0 `vec_dot` / `from_float` location

- **Not here.** `ggml_get_type_traits_cpu` / `type_traits_cpu[]` in **`ggml-cpu.c`** set `vec_dot` to `ggml_vec_dot_q8_0_q8_0` and `from_float` for packed rows used when quantizing activations to Q8 in generic mul_mat / repack forward.

- **Bypass map:** any `compute_forward` that returns true hides downstream hooks unless the trait itself calls them (repack does not use gyro hooks per export 8).

# Export 12: `external/llama.cpp/tools/llama-bench/llama-bench.cpp`

## Purpose

Batch benchmark driver: loads **`llama.h`**, builds contexts from CLI grids, runs **prompt** and **generation** timing tests, prints **CSV / JSON / JSONL / MD / SQL** rows with **tokens/s** (`avg_ts`, `stddev_ts`) and **nanoseconds** (`avg_ns`, `stddev_ns`) per configuration.

## CPU-only, single-threaded (typical flags)

- **`-m` / `--model`** `<path.gguf>` -- model file.
- **`-t` / `--threads`** `1` -- math threads (also passed to `llama_set_n_threads`).
- **`-ngl` / `--n-gpu-layers`** `0` -- no GPU offload (CPU backend for weights).
- Optional tighten: **`-p` / `--n-prompt`**, **`-n` / `--n-gen`** token counts; **`-b` / `--batch-size`**, **`-ub` / `--ubatch-size`**; **`-fa 0`** to disable flash-attn if comparing kernels.

Full list: **`print_usage`** in this file (~398+).

## Output

- **`-o csv`** (default class of tools) -- columns include `model_filename`, `n_threads`, `n_gpu_layers`, `n_prompt`, `n_gen`, `avg_ts`, `stddev_ts`, `avg_ns`, `stddev_ns`, `test_time`, backend info, etc. (`bench_result::get_fields` ~1451+).
- Parse **numeric** fields for regression; compare **same** `-m`, `-p`, `-n`, batch settings between runs.

## `GGML_GYROSCOPIC_*` env

Not referenced in this source; flags are read by **ggml / llama** after `llama_backend_init()`. Any binary linked against **gyroscopic** ggml honors **`GGML_GYROSCOPIC_MATMUL`**, **`GGML_GYROSCOPIC_STRICT`**, **`GGML_GYROSCOPIC_TRACE`**, **`GGML_GYROSCOPIC_KERNEL`** the same as **`llama-cli`**.

## Example command lines (PowerShell)

**Stock matmul path (hooks off if `GGML_GYROSCOPIC_MATMUL` unset or 0):**

```text
.\llama-bench.exe -m "F:\path\Qwen3.5-4B-Q8_0.gguf" -t 1 -ngl 0 -o csv
```

**Gyro matmul on (compare throughput + stderr trace):**

```text
$env:GGML_GYROSCOPIC_MATMUL="1"; $env:GGML_GYROSCOPIC_TRACE="1"; .\llama-bench.exe -m "F:\path\Qwen3.5-4B-Q8_0.gguf" -t 1 -ngl 0 -o csv
```

Unset env vars between runs or open a fresh shell for the baseline.

## What to compare

- **`avg_ts` / `stddev_ts`** (tokens/s) for identical **`-p`**, **`-n`**, **`-b`**, **`-ub`**, **`-fa`**, model path.
- With **`GGML_GYROSCOPIC_TRACE=1`**, stderr **hook counters** vs stock (should move when gyro path fires).

# Export 17: `external/llama.cpp/ggml/src/ggml-gyroscopic/binary-ops.cpp`

## Ops implemented (elementwise, non-quantized)

Each forwards to **`binary_op<op_*>`** with dtype dispatch (F32/F16/BF16 combos including mixed F32 broadcast src1):

- **`ggml_compute_forward_add_non_quantized`** -- `op_add`
- **`ggml_compute_forward_sub`** -- `op_sub`
- **`ggml_compute_forward_mul`** -- `op_mul` (tensor mul, **not** `GGML_OP_MUL_MAT`)
- **`ggml_compute_forward_div`** -- `op_div`

## Mechanism

- Per-element: cast inputs to **float**, apply `op(a,b)`, cast to destination type via **`type_conversion_table`** (`common.h`).
- Threading: **`get_thread_range`**, row loops; contiguous vs broadcast **`src1`** paths (`vec_binary_op_contiguous` / `_non_contiguous`).
- **macOS Accelerate:** when all tensors **F32**, optional **vDSP** (`vDSP_vadd`, `vDSP_vsub`, `vDSP_vmul`, `vDSP_vdiv`) for contiguous segments.

## Bitwise integer ops (AND / OR / XOR on tensors)

**None.** All arithmetic is **float-valued** through **`op_add`**, **`op_mul`**, etc. Chirality-style **integer bitwise** in-graph would require **new ops** and kernels, not an extension of this file.

## Gyroscopic action

- **Bypass:** No interaction with matmul hooks; standard elementwise surface.
- **log.md:** Record **ADD/SUB/MUL/DIV** non-quantized live here; **MUL_MAT** remains **`ggml-cpu.c`** / **`ops.cpp`** stack.

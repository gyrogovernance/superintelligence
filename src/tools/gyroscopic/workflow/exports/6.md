# Export 6: `external/llama.cpp/ggml/src/ggml-gyroscopic/quants.c` (top-level)

## Role vs arch TU

This file is the **portable reference layer**: scalar dot loops, delegations to `*_ref` quantizers, and shared dot-product **`*_generic`** bodies. It does **not** contain the x86 AVX2 `ggml_vec_dot_q8_0_q8_0` entry point; that symbol is defined in **`arch/x86/quants.c`** (`exports/5.md`). The linker picks one `quantize_row_q8_0` / `ggml_vec_dot_q8_0_q8_0` per arch object file.

## How `ggml_vec_dot_q8_0_q8_0` is selected (not a runtime table)

- **Preprocessor + link time**, not a function-pointer table inside this file.
- **`arch-fallback.h`** (included after `simd-mappings.h`): under **`GGML_CPU_GENERIC`**, `#define ggml_vec_dot_q8_0_q8_0_generic ggml_vec_dot_q8_0_q8_0` so the generic body becomes the public symbol.
- Under **x86** (`__x86_64__` / `_M_X64` etc.), that alias is **not** applied for `q8_0_q8_0`; the TU **`arch/x86/quants.c`** supplies **`void ggml_vec_dot_q8_0_q8_0(...)`** while **`ggml_vec_dot_q8_0_q8_0_generic`** remains a separate reference symbol in this file.

## `block_q8_0` layout (header, not this .c)

Defined in **`ggml/src/ggml-common.h`** (via `ggml-common.h` include chain):

- `QK8_0` = **32**
- `typedef struct { ggml_half d; int8_t qs[QK8_0]; } block_q8_0;`
- **32-wide** quant lane per block; natural **64-wide fusion** = two consecutive blocks (grain note for codec).

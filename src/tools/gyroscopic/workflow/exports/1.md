## Local structs

**`input_tensor`**

- Fields: `ggml_type type`; `std::array<int64_t,4> ne`; `std::array<size_t,4> nb`.
- `operator<`: lexicographic `std::tie(type, ne, nb)`.
- `serialize`: emits `type` then four `ne` then four `nb`, space-separated (trailing space after last nb before parent continues).

**`test_object`**

- Fields: `ggml_op op`; `ggml_type type`; `std::array<int64_t,4> ne`; `std::vector<int32_t> op_params`; `std::vector<input_tensor> sources`; `std::string name`.
- `operator<`: `std::tie(op, type, ne, op_params, sources)` -- **`name` is not part of the ordering key.**
- Consequence for `std::set<test_object>`: two nodes that differ only by `node->name` (or any field not in the tie) are **one** exported row; `set` keeps the first inserted element’s `name` and drops the other’s label.
- `serialize` line format (space-separated tokens, newline-terminated):
  1. `op` via `ostream <<` (C `enum ggml_op` promoted to integer in practice).
  2. `type` via `ostream <<` (C `enum ggml_type` promoted to integer).
  3. `ne[0..3]`.
  4. `op_params.size()` then that many `int32` values (always resized to `GGML_MAX_OP_PARAMS / sizeof(int32_t)` before copy).
  5. `sources.size()` then for each source: `type`, four `ne`, four `nb` (via `input_tensor::serialize`).
  6. If `name` non-empty: raw `name` string (no delimiter escaping). Else literal `-`.
  7. `\n`.

**Constants from ggml (verified `ggml/include/ggml.h`)**

- `GGML_MAX_SRC` = 10.
- `GGML_MAX_OP_PARAMS` = 64 (bytes); exporter uses `GGML_MAX_OP_PARAMS / sizeof(int32_t)` = **16** int32 words per node.

## `extract_graph_ops(cgraph, label, tests)`

- Iterates `i in [0, ggml_graph_n_nodes(cgraph))`, node = `ggml_graph_node(cgraph, i)`.
- **Skipped ops** (not inserted): `GGML_OP_NONE`, `GGML_OP_VIEW`, `GGML_OP_RESHAPE`, `GGML_OP_PERMUTE`, `GGML_OP_TRANSPOSE`. All other ops in the graph are candidates.
- For each kept node: copies `op`, `type`, first four `ne`; `op_params` vector sized to 16 ints, `memcpy` **64 bytes** from `node->op_params`; sources = consecutive `node->src[s]` until first `nullptr` (order preserved).
- `LOG_INF("%s: %d unique ops, %d total nodes, %d skipped (view ops)\n", label, n_new, n_nodes, n_skipped)` where `n_new` is growth of `tests.size()` for this pass (new unique keys vs set before this graph).

## `main`

**CLI**

- `common_params_parse(argc, argv, params, LLAMA_EXAMPLE_EXPORT_GRAPH_OPS)` (`common/arg.h`).
- Default `params.out_file = "tests.txt"` before parse (overridable by `-o` / `--output` / `--output-file` -- see `common/arg.cpp` around the option registered with `.set_examples({..., LLAMA_EXAMPLE_EXPORT_GRAPH_OPS})`).
- All options that apply to `LLAMA_EXAMPLE_COMMON` in `common_params_parser_init` also apply; this TU does not add example-specific flags beyond shared output file wiring.

**Init**

- `common_init()`.
- Forces **CPU-only**: `ggml_backend_dev_by_type(GGML_BACKEND_DEVICE_TYPE_CPU)`; `params.devices = { cpu_device, nullptr }`; `params.n_gpu_layers = 0`; `params.fit_params = false`; `params.warmup = false`.
- `common_init_from_params(params)` -> `init_result->context()` as `llama_context * ctx`. **No null check** on `ctx` in this TU; failed init would dereference null on following lines.

**Shape parameters**

- `n_seqs = llama_n_seq_max(ctx)` (`include/llama.h`).
- `n_tokens = std::min(llama_n_ctx(ctx), llama_n_ubatch(ctx))`.

**Graphs**

- PP: `gf_pp = llama_graph_reserve(ctx, n_tokens, n_seqs, n_tokens)`; throws if null.
- `extract_graph_ops(gf_pp, "pp", tests)`.
- TG: `gf_tg = llama_graph_reserve(ctx, n_seqs, n_seqs, n_seqs)`; throws if null.
- `extract_graph_ops(gf_tg, "tg", tests)`.
- Second `llama_graph_reserve` invalidates the **previous** reserved graph pointer per API comment on `llama_graph_reserve` in `src/llama-ext.h`; this program only uses `gf_pp` before the second call, so ordering is safe.

**Output**

- `LOG_INF("%d unique ops total\n", (int) tests.size())`.
- Opens `params.out_file`; throws if not open.
- Writes `test.serialize` for each element of `tests` in **sorted order** (`std::set` iteration).

**No** `graph_compute`, no token loop, no sampling.

## Lead: `llama_graph_reserve` (read for graph semantics)

- Declared `src/llama-ext.h`: allocates/returns `ggml_cgraph *`; valid until next `llama_graph_reserve` on same context.
- Defined `src/llama-context.cpp`: obtains `mctx` from `ctx->get_memory()->init_full()` when memory exists; calls `ctx->graph_reserve(n_tokens, n_seqs, n_outputs, mctx.get())` with default `split_only = false` (see `src/llama-context.h`).

## Lead: `llama_context::graph_reserve` (where `build_graph` runs)

- `src/llama-context.h`: `ggml_cgraph * graph_reserve(uint32_t n_tokens, uint32_t n_seqs, uint32_t n_outputs, const llama_memory_context_i * mctx, bool split_only = false, size_t * sizes = nullptr)`.
- `src/llama-context.cpp` implementation (representative sequence): if `n_tokens % n_seqs != 0`, rounds `n_tokens` up to a multiple of `n_seqs` and may bump `n_outputs`; `ggml_backend_sched_reset`; synthetic `llama_ubatch` via `llama_batch_allocr::ubatch_reserve(n_tokens/n_seqs, n_seqs)`; marks one output per sequence; `graph_params(...)`; `res->reset()`; **`gf = model.build_graph(gparams)`**; restores `n_outputs`; **`ggml_backend_sched_reserve(sched, gf)`** (when not `split_only`); returns `gf`.

**Material elsewhere:** node **kinds and topology** come from `llama_model::build_graph` and arch-specific `llm_build_*` under `src/models/*.cpp` plus shared helpers in `src/llama-graph.cpp`. This exporter does not read those files; it only consumes the resulting `ggml_cgraph`.

## Lead: `llama_batch_allocr::ubatch_reserve`

- `src/llama-batch.cpp`: builds `llama_ubatch` with `n_tokens = n_seq_tokens * n_seqs`, `b_equal_seqs = true`, allocates pos/token/seq_id/output arrays. Used to shape the dummy batch for reserve.

## Playbook buckets (this TU only)

- **Op / API contract:** indirect; reflects serialized `ggml_tensor` node view (op, type, ne, op_params, src types/ne/nb), not full ggml runtime contracts.
- **Dispatch path:** none; no `ggml_compute_forward_*`.
- **Orchestration:** none.
- **Microkernel grammar:** none.
- **Repack / bypass:** none.
- **Hook locus:** none.
- **Validation:** yes -- inventory of which ops and static shapes appear in reserved PP/TG graphs for the loaded GGUF and context params.

## Limits of the artifact (what consumers must not infer)

- File lines are **not** execution order, scheduler order, or “which repack path runs.”
- Integer `op`/`type` values depend on **header** enum numeric values at build time; parsers should map via `ggml` headers or `ggml_op_name` / `ggml_type_name` if they translate externally.
- **Not all layout ops are skipped:** only the five listed; e.g. `GGML_OP_CPY`, `DUP`, etc. would appear if present.
- **Name field** unreliable for uniqueness (dedup ignores name).
- **CPU-only** device list: graph may differ from GPU-split builds for the same GGUF.
- **op_params** always dumps 16 int32s per line even when the op uses fewer; unused words are still copied from tensor storage.

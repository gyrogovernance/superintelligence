# Export 3: `external/llama.cpp/ggml/src/ggml-gyroscopic/ggml-cpu.c`

## Entry and bypass (before `switch`)

1. `ggml_compute_forward(params, tensor)`: if `GGML_OP_NONE` or empty, return.
2. **`if (ggml_cpu_extra_compute_forward(params, tensor)) return;`** (`traits.cpp`): iterates registered extra buffer types; if `tensor_traits->compute_forward` handles the op, **entire generic path skipped** (repack / AMX / etc. per build). **Bypass risk** for MUL_MAT when tensors use those buffers.

## `type_traits_cpu` / vec_dot routing

- Static table `type_traits_cpu[GGML_TYPE_COUNT]` (~212+): per `src0` type sets `.vec_dot` (e.g. `ggml_vec_dot_q8_0_q8_0` for `GGML_TYPE_Q8_0`) and `.vec_dot_type` (often `GGML_TYPE_Q8_0` for quant dots), plus `.from_float`, `.nrows`, etc.
- `ggml_get_type_traits_cpu` accessor (~403).

## `ggml_compute_forward_mul_mat` call chain

**A. Optional llamafile** (`#if GGML_USE_LLAMAFILE`): may take early return after `llamafile_sgemm`; label `UseGgmlGemm1`.

**B. Quantize activations (`src1`) to `vec_dot_type`**

- If `src1->type != vec_dot_type`: requires `src1 == F32`, uses `params->wdata`, `from_float` in parallel strips over `ne10` blocks (`ith`/`nth` split). Workspace layout: `nbw1 = row_size(vec_dot_type, ne10)`, `nbw2`, `nbw3`; `params->wsize >= ne13*nbw3`.

**C. Gyroscopic fast path** (`#ifdef GGML_USE_GYROSCOPIC`, ~1346-1403)

- **When:** `ggml_gyroscopic_active()` AND `vec_dot_type == Q8_0` AND `src0->type == Q8_0` AND `dst->type == F32` AND `(ne00 % 32) == 0` AND `params->threadpool != NULL`.
- **Thread 0 only** validates and loops `(i13,i12)` broadcast slices; calls `ggml_gyroscopic_gemm_q8_0_q8_0(M,N,K, A, nb01, B, ldb_bytes, C, ldc)` per slice. Failure sets `g_gyro_mul_mat_done` false.
- **Barrier**; if atomic success, **return** (skips vec_dot chunk loop).
- **Bypass relative to vec_dot:** this path never calls `ggml_vec_dot_q8_0_q8_0`; it uses bridge GEMM directly.

**D. Optional llamafile path 2** (`UseGgmlGemm2`) when `src1` not `vec_dot_type` after conversion.

**E. Chunked parallel loop (stock / fallback)**

- `nr0 = ne0`, `nr1 = ne1*ne2*ne3`, `chunk_size` 16 or 64 if `nr0==1 || nr1==1`.
- `nchunk0`, `nchunk1`; if `nchunk0*nchunk1 < nth*4` or NUMA, reassign chunks to threads.
- Work-stealing via `params->threadpool->current_chunk` (initialized to `nth` at barrier from `ith==0`).
- **`num_rows_per_vec_dot = vec_dot_num_rows`** from traits; forced to **1** if parity checks on `nr0`, `ne11`, chunk bounds fail (avoids 2-row kernel crossing dim1 boundaries).
- **`ggml_compute_forward_mul_mat_one_chunk`** with `(ir0_start, ir0_end, ir1_start, ir1_end)`.

### `ggml_compute_forward_mul_mat_one_chunk`

- `vec_dot = type_traits_cpu[type].vec_dot`, `vec_dot_type` from traits.
- `wdata = (src1->type == vec_dot_type) ? src1->data : params->wdata`.
- Tile `blck_0 = 16`, `blck_1 = 16`; `tmp[32]` float buffer.
- Inner call: **`vec_dot(ne00, tmp, ..., src0_row, nb01, src1_col, src1_col_stride, num_rows_per_vec_dot)`** -> for Q8_0 x Q8_0 hits **`ggml_vec_dot_q8_0_q8_0`** (hooked in x86 quants).

## `ggml_compute_forward_mul_mat_id`

- Similar `vec_dot_type` / `from_float` / `wdata` quant path; **no `#ifdef GGML_USE_GYROSCOPIC` GEMM block** in the scanned region -> **MUL_MAT_ID always uses chunk + vec_dot** (or extra forward) for Q8 paths unless repack intercepts.
- `ggml_compute_forward_mul_mat_id_one_chunk`: `blck_0/blck_1 = 16`, `vec_dot(..., nrc=1)`.

## OUT_PROD

- Switch dispatches to `ggml_compute_forward_out_prod` in `ops.cpp` (see export 2).

## Workspace (`ggml_graph_work_size` area)

- Cases `MUL_MAT` / `MUL_MAT_ID` (~2861+): size includes row pack of `src1` into `vec_dot_type` when types differ (`ggml_row_size(vec_dot_type, nelements)` + alignment); mul_mat_id adds `sizeof(int64_t)` in one branch.

## Threading

- `ggml_barrier(params->threadpool)` between gyro decision and chunk loop; atomic chunk counter for parallel distribution.

## Playbook buckets

- **Dispatch path:** extra -> gyro GEMM -> llamafile -> chunked vec_dot.
- **Repack bypass:** `ggml_cpu_extra_compute_forward` true => generic mul_mat body not run.
- **Orchestration:** chunk_size / nchunk / `num_rows_per_vec_dot` gating.

## Exactness (MUL_MAT Q8)

- Gyro GEMM / vec_dot hooks: **deterministic-numeric** (integer dot core + fp16 scales + float acc) per bridge spec.
- `from_float` path: **float** activations converted to Q8 layout.

# Export 7: `external/llama.cpp/ggml/src/ggml-gyroscopic/simd-gemm.h`

## What it is

Header-only **float32 GEMM** micro-tiling: `C[M x N] += A[M x K] * B[K x N]`. Uses **`GGML_F32_VEC_*`** from **`simd-mappings.h`**; active when **`GGML_SIMD`** is defined and not SVE / RISC-V vector-only branch inside this header.

## Tile / register constants (compile-time)

From the `#if` ladder:

- **AVX512F or ARM NEON (in this file):** `GEMM_RM = 4`, `GEMM_RN = 4` (comment: 16+4+1 = 25/32 ymm lanes metaphor).
- **AVX2 / AVX (typical Windows x86):** `GEMM_RM = 6`, `GEMM_RN = 2` (comment: 12+2+1 = 15/16).
- **Else:** `GEMM_RM = 2`, `GEMM_RN = 2`.

**K tile:** no fixed `K_tile`; inner loop runs **`kk = 0 .. K-1`** inside `simd_gemm_ukernel` (full K depth per micro-tile).

**N blocking:** uses **`KN = GGML_F32_EPR`** (floats per vector, **8** on AVX2). Outer `simd_gemm` steps **`jj` by `GEMM_RN * KN`** (e.g. **16** columns per wide step when `GEMM_RN=2`, `KN=8`), then tail with `GEMM_RN,1` then scalar column tails.

## Microkernel `simd_gemm_ukernel<RM, RN>`

- **Accumulators:** `GGML_F32_VEC acc[RM][RN]` -- **RM*RN** vector registers of width `GGML_F32_EPR`.
- **Load C** as initial accumulators (fused add-mul pattern).
- **Inner k loop:** for each `kk`, load **`RN`** vectors from B row `kk`, broadcast scalar **`A[i*K+kk]`** per row `i`, **FMA** into `acc[i][r]`.
- **Store** `acc` back to C.

**Per k iteration:** **RN** B loads + **RM** broadcasts + **RM*RN** FMAs (all **float**).

## `simd_gemm` driver

- Strip-mines **M** by **`GEMM_RM`**, **N** by **`GEMM_RN * KN`** with tails; advances **`A`**, **`C`** pointers by processed rows.

## Connection to `MUL_MAT` (Q8_0)

- **Not** the generic `ggml_compute_forward_mul_mat` Q8 path (that uses **vec_dot** / **gyro GEMM** on quantized blocks per `ggml-cpu.c`).
- **Used from `ops.cpp`** when **`GGML_SIMD`** and **`#include "simd-gemm.h"`** -- e.g. **flash-attention style** paths: **`simd_gemm(KQ, Q_q, K_f32, ...)`** and **`simd_gemm(VKQ32, KQ, V32, ...)`** on **packed float** tiles (Q/K/V after dequant or f32), not on raw `block_q8_0` tensors.

## Q8_0 compatibility

- This header is **F32-only**; **no Q8_0 block awareness**. Quantized matmul grammar remains **quants.c / repack / gyro bridge**.

## Exactness

- **Float** path end-to-end (FMA on `float`).

## Bypass risk vs gyro hooks

- **Different surface:** F32 attention GEMM in **`ops.cpp`** does **not** go through **`ggml_vec_dot_q8_0_q8_0`** or **`ggml_vec_dot_f32`** for those tile calls. If you need gyro coverage on that subgraph, you must **instrument `simd_gemm` call sites** or the surrounding op, not only vec_dot hooks.

## Gyroscopic action

- **codec.c:** Borrow **RM/RN/N-vector striping** and **register-block FMA** layout for a **float or dequanted** staging GEMM; for **native Q8** use export 5 / repack patterns instead.
- **Hooks:** Optional future `#ifdef GGML_USE_GYROSCOPIC` wrapper around **`simd_gemm`** or its call sites in **`ops.cpp`** if attention must be owned.
- **log.md:** Note **simd_gemm** = **F32 flash-attn helper**, orthogonal to Q8_0 weight `MUL_MAT` vec_dot path.

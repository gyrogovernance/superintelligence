# Export 10: `external/llama.cpp/ggml/src/ggml-gyroscopic/vec.cpp`

## `ggml_vec_dot_f32` -- gyro hook + stock x86 SIMD

**Hook (first):** if `GGML_USE_GYROSCOPIC` and `ggml_gyroscopic_active()`, call `ggml_gyroscopic_vec_dot_f32(n, x, y, s)`; on success return. If strict and bridge failed, `ggml_gyroscopic_abort_unsupported("ggml_vec_dot_f32", F32, F32)`.

**Stock path (non-SVE / non-RISCV branch, typical x86):** under `GGML_SIMD`, main loop steps by **`GGML_F32_STEP`** with **`GGML_F32_ARR`** lanes of **`GGML_F32_EPR`**-wide vectors: load `ax[j]`, `ay[j]`, **`GGML_F32_VEC_FMA(sum[j], ax[j], ay[j])`**, then **`GGML_F32_VEC_REDUCE(sumf, sum)`**, then scalar tail `for (i = np; i < n) sumf += x[i]*y[i]`. **Float** accumulation (**float** exactness class).

**Scalar fallback:** `ggml_float` loop when no `GGML_SIMD`.

**Callers (conceptual):** any path that uses **`type_traits_cpu[GGML_TYPE_F32].vec_dot`** for dot-like work, including **F32 `MUL_MAT`** inner loops in `ggml-cpu.c` when both operands are F32. Not used for Q8_0 matmul (that uses `ggml_vec_dot_q8_0_q8_0`).

## Other `ggml_vec_*` in this file (one line each)

- **`ggml_vec_dot_bf16`** -- BF16 dot; AVX512 BF16 / AVX512F widen / AVX2 widen+mul paths + scalar tail.
- **`ggml_vec_dot_f16`** -- F16 dot; wide SIMD paths + scalar; no gyro hook in scanned code.
- **`ggml_vec_silu_f32`** -- in-place SILU; AVX512 / AVX2 / SSE2 / ARM / RISC-V chunks + scalar tail. **Activation hot path.**
- **`ggml_vec_swiglu_f32`** -- `silu(x) * g`; same ISA ladder. **FFN-style hot path.**
- **`ggml_vec_cvar_f32`** -- `(x-mean)^2` into `y`, returns mean variance; AVX-style horizontal adds. **Norm / stats.**
- **`ggml_vec_soft_max_f32`** -- `exp(x-max)` into `y`, returns sum of exponentials. **Attention softmax building block** (used with tile logic in `ops.cpp` flash paths).
- **`ggml_vec_log_soft_max_f32`** -- scalar-only loop: stores `x[i]-max`, returns `log(sum exp)`; no AVX2 fast path in this function body.

**Note:** `ggml_vec_add_f32`, `ggml_vec_scale_f32`, `ggml_vec_max_f32`, etc. are **not** defined in this `.cpp`; they live in **`vec.h`** as inlines / overloads (flash-attn and other ops include that header).

## Gyroscopic action

- **Hook:** Only **`ggml_vec_dot_f32`** is wired to the bridge today; extend other dots only if F16/BF16 matmul ownership is required.
- **Bypass:** Q8_0 matmul does not hit this symbol; **F32 matmul** and any op using F32 `vec_dot` does. **`simd_gemm`** in `ops.cpp` bypasses vec_dot for those F32 tile GEMMs (`exports/7.md`).
- **log.md:** F32 dot hook location = start of `ggml_vec_dot_f32` after asserts.

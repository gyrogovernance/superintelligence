# Export 15: `external/llama.cpp/tests/test-quantize-fns.cpp`

## What it tests

Unit tests for **quantize / dequant / vec_dot** per **`ggml_type`**: uses **`ggml_get_type_traits`** and **`ggml_get_type_traits_cpu`**, **`from_float`**, **`to_float`**, and **`qfns_cpu->vec_dot`** via **`dot_product_error`** (quantizes two synthetic float vectors, calls **`vec_dot`**, compares to float reference dot).

## Q8_0 dot

- **`GGML_TYPE_Q8_0`** is included in the **`for (i < GGML_TYPE_COUNT)`** loop whenever **`from_float`**, **`to_float`**, and **`vec_dot`** are non-null. Dot threshold uses **`MAX_DOT_PRODUCT_ERROR`** (**0.02f**) unless a low-bit branch applies (Q8_0 uses the default).

## Gyroscopic backend

- Calls **`ggml_cpu_init()`**; uses **CPU type traits** from whatever **`ggml-cpu`** / backend the test binary was linked with. For gyro hooks to execute, **`vec_dot`** for Q8_0 must resolve to the **gyroscopic** TU and **`GGML_GYROSCOPIC_MATMUL`** must allow the bridge without strict abort on this synthetic shape (verify env when running).

## What passing proves

- **Quantized dot + round-trip** for that type match expectations vs float reference within test tolerances; catches gross **vec_dot** regressions when the hooked symbol is active.

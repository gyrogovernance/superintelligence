# Export 13: `external/llama.cpp/tools/perplexity/perplexity.cpp`

## Purpose

Computes **perplexity**, optional **HellaSwag / Winogrande / multiple-choice / KL** modes via **`common_params_parse(..., LLAMA_EXAMPLE_PERPLEXITY)`**. Uses **`llama.h`**: **`common_init_from_params`**, **`perplexity(ctx, params, n_ctx)`** (default branch), then **`llama_perf_context_print`** and **`llama_memory_breakdown_print`**.

## CPU-only, single-threaded (typical)

Uses **shared** `common` CLI (see **`common/arg.cpp`** for full list). Typical minimal shape:

- **`-m` / `--model`** `<path.gguf>`
- **`-t` / `--threads`** `1`
- **`-ngl` / `--n-gpu-layers`** `0`
- **`-c` / `--ctx-size`** `512` (tool sets default **512** in `main` if not overridden; must be **> 0**)
- **`-f` / `--file`** text corpus for tokenized perplexity (or other data flags per mode)

**Note:** Tool adjusts **`n_parallel`** from **`n_batch`** and **`n_ctx`** (~2031-2035); keep **`n_batch`** sane vs context for stable runs.

## Output

- **LOG**-style lines: progress, final **perplexity** (`ppl_value` / printed summary in `perplexity()`), plus **`llama_perf_context_print`** (**prompt eval t/s**, **eval t/s**, etc.).
- Compare **reported PPL** (should match within float noise if hooks are numerically faithful) and **perf** lines between stock vs gyro env.
